# ТЗ — АИС «eSport»
**Заказчик:** Государственное агентство физической культуры и спорта (ГАФКиС)  
**Версия:** 1.0 | **Дата:** 05.12.2025

---

## 1. ОБЩИЕ СВЕДЕНИЯ
- **Название системы:** АИС «eSport» — автоматизированная информационная система управления процессами в сфере профессионального и любительского спорта
- **Пользователи:** сотрудники ГАФКиС (1000), внешние пользователи (10 000), супер-администратор (1), системный администратор (1)
- **Стек технологий:** C# .NET / Java Spring Boot / Golang (бэкенд), ReactJS / VueJS / NodeJS (фронтенд), Flutter (мобильное), PostgreSQL (БД)

---

## 2. МОДУЛИ СИСТЕМЫ

### 2.1 Модуль «Внешний портал (веб-сайт)»
Публичный сайт ГАФКиС для внешних пользователей.

**Разделы сайта:**
- Главная (сводная информация)
- Новости
- О ГАФКиС (структура, руководство, контакты, пресс-служба, областные управления, спортшколы)
- НПА (законы, постановления, реестры)
- Программный бюджет
- Календарный план мероприятий
- Спортивные школы
- Спортсмены (профили, международный рейтинг)
- Тренеры
- Сборные команды
- Судьи
- Виды спорта
- Спортивные объекты (карта, WMS геосервис)
- Объявления (вакансии, тендеры, аренда)
- Общественные обсуждения
- Антикоррупционные меры
- Государственные услуги
- Антидопинговая деятельность
- Проверка подлинности документа по QR-коду
- Личный кабинет (спортсмен / тренер / судья)

**Личный кабинет:**
- Спортсмен: паспорт, медсправка, мероприятия, тренировки, тренеры, медали, сборная
- Тренер: сертификат, спортсмены, мероприятия, тренировки, сборная
- Судья: категория, мероприятия
- Онлайн подача заявлений (спортзвание, тренерский сертификат, судейская категория, восстановление документов)

**Интеграции:**
- СМЭВ «Түндүк» ЕСИ — авторизация пользователей
- Внутренний портал — отображение данных

**Требования к дизайну:**
- Адаптивный (мобильные, планшеты)
- Два языка: кыргызский (по умолчанию) + русский, опционально английский
- Единый стиль всех страниц
- Версия для слабовидящих (контраст не менее 4.5:1)
- Быстрая навигация (не более 3 кликов до любой страницы)
- Хедер: логотип, навигация, авторизация, выбор языка

### 2.2 Модуль «Внутренний портал»
Внутренняя система для сотрудников ГАФКиС.

#### База данных eSport — реестры:

**Реестр спортсменов** (паспорт спортсмена):
- ФИО, дата рождения, пол, фото
- Контакты (телефон, email)
- Спортивное звание / разряд
- Медицинские документы
- Участие в мероприятиях
- Тренировочный процесс
- Тренер, сборная команда
- Медали и достижения
- Скан-копии документов
- Визы, аккредитации, членства в федерациях
- Инвентарь, страхование

**Реестр тренеров:**
- ФИО, контакты, фото
- Номер и срок лицензии
- Категории / специализации
- Привязанные организации, тип занятости

**Реестр судей:**
- ФИО, контакты
- Номер и срок судейского удостоверения
- Категории, назначенные мероприятия

**Реестр сотрудников:**
- ФИО, должность, подразделение
- Дата приёма, статус (активен / уволен / отпуск)
- Роли доступа, привязка к AD/ЕСИ

**Реестр спортивных организаций:**
- Название, тип (федерация / клуб / учебное заведение)
- ИНН, контакты, регион
- Связанные спортсмены, тренеры

**Реестр спортивных объектов:**
- Название, адрес, геокоординаты
- Вместимость, оснащение
- Расписание доступности

**Реестр мероприятий / событий:**
- Название, тип, даты, место
- Участники (спортсмены, команды)
- Результаты, статус

**Реестр сборных команд:**
- Название, возрастная категория
- Тренер, состав, связанные мероприятия

**Реестр медицинских справок:**
- Кто выдал, дата выдачи, срок действия
- Тип, диагноз (ICD10), скан, статус верификации

**Реестр инвентаря:**
- Название, код, категория, количество
- Состояние, местоположение, кому назначен

#### Интранет (внутренний портал):
- Новости для сотрудников
- Структура ГАФКиС
- Справочник сотрудников с контактами
- Регламенты, инструкции, приказы
- База знаний, шаблоны документов
- Внутренний чат
- Объявления (HR, IT, руководство)
- Календарь событий

#### Онлайн заявления:
- Приём и обработка заявлений по настроенным маршрутам
- Совместная работа, согласование
- Импорт документов из СМЭВ «Түндүк»
- Результат: отклонение или выдача документа онлайн

#### Согласование документов:
- Внутренние документы по маршрутам (инициатор — сотрудник)
- Примеры: заявки на закупку, заявления на отпуск

#### Маршруты:
- Настройка неограниченного количества маршрутов
- Этапы: согласование (руководитель) или прикрепление документа (исполнитель)

#### Аналитические отчёты:
- Определяются в ходе реализации

**Интеграции внутреннего портала:**
- СМЭВ «Түндүк» — авторизация, импорт документов заявителей по ИНН, экспорт спортдокументов
- АИС Е-Кызмат — импорт данных спортсменов, тренеров, судей, сотрудников
- Министерство здравоохранения — импорт данных медсправок по ИНН

### 2.3 Модуль «Администрирование»
- Изменение внутренних параметров системы
- Резервное копирование
- Регистрация пользователей и разграничение прав доступа

---

### 2.4 Модуль «Управление доступами»
- Ведение списка пользователей (добавление, блокировка, разблокировка)
- Назначение паролей и ролей
- Разграничение прав на уровне региональных управлений
- История изменений прав доступа
- Ролевая модель: просмотр / редактирование
- Блокировка после N неуспешных входов
- Автоблокировка по таймауту неактивности
- Журнал действий пользователей

---

### 2.5 Модуль «Интеграция»
- СМЭВ «Түндүк» ЕСИ — авторизация
- СМЭВ «Түндүк» — импорт документов по ИНН
- СМЭВ «Түндүк» — экспорт спортивных документов (приложение Түндүк)
- АИС Е-Кызмат — импорт данных участников
- Министерство здравоохранения — медсправки
- WMS геосервис — карта спортивных объектов

## 3. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### Архитектура:
- Трёхуровневая клиент-серверная (UI → бизнес-логика → данные)
- Централизованное хранилище данных
- Многопользовательский доступ
- REST API для интеграций
- Поддержка кластеризации СУБД

### Производительность:
- Загрузка страницы — до 10 секунд
- Генерация отчётов — до 1 минуты
- Восстановление из резервной копии — не более 4 часов

### Безопасность:
- Аутентификация по логину/паролю и/или Active Directory
- «Слепые» пароли
- Журнал событий безопасности
- Шифрование, антивирус
- Соответствие постановлениям КР №760 и №762 от 21.11.2017

### БД:
- PostgreSQL (реляционная)
- Ссылочная целостность, резервное копирование
- Хранение документов в форматах XML / JSON

### Браузеры:
- Все современные браузеры, без проприетарных плагинов
- Кодировка UTF-8
