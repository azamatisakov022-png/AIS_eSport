## 1. Общие сведения

### 1.1 Наименование системы

Полное: **Автоматизированная информационная система электронного учёта в сфере физической культуры и спорта Кыргызской Республики**
Сокращённое: **АИС eSport**

### 1.2 Основание для разработки

Система разрабатывается для автоматизации учёта спортсменов, тренеров, судей, спортивных организаций, мероприятий, спортивных объектов и обработки заявлений граждан в сфере физической культуры и спорта КР.

## 2. Назначение и цели системы

### 2.1 Назначение

Единая цифровая платформа для:
- Ведения реестров спортсменов, тренеров, судей, организаций, спортивных объектов
- Управления спортивными мероприятиями и их результатами
- Обработки заявлений на присвоение спортивных званий и регистрацию тренеров
- Предоставления публичного доступа к открытым данным в сфере спорта
- Аналитики и отчётности по показателям спортивной деятельности

### 2.2 Цели

1. **Цифровизация** — перевод бумажного документооборота ГАФКиС в электронный формат
2. **Прозрачность** — открытый доступ к реестрам и верификации документов
3. **Эффективность** — сокращение сроков обработки заявлений, автоматический контроль дедлайнов
4. **Аналитика** — оперативный мониторинг KPI в сфере спорта по регионам, видам спорта, годам
5. **Единый реестр** — централизованное хранение данных о всех субъектах спортивной сферы КР

**Состав:**

| **Государственные услуги** | 3 карточки с описанием услуг и регламентных сроков: Регистрация тренера, Заявка на звание, Категория судьи |

#### 5.1.2 Публичные реестры

Открытые реестры доступны без авторизации.

**Реестр спортсменов** (`/public/athletes`):
- Карточный список с фильтрацией: поиск по имени, вид спорта, разряд, регион
- Пагинация (12 карточек на странице)
- Карточка: аватар, ФИО, вид спорта, разряд, регион, медали (золото/серебро/бронза)
- Переход на профиль спортсмена (`/public/athletes/:id`)

**Профиль спортсмена** (`/public/athletes/:id`):
- Личные данные, вид спорта, разряд
- История разрядов и званий
- Медали и достижения
- Команда, тренер

**Реестр тренеров** (`/public/coaches`):
- Список с фильтрацией: поиск, вид спорта, регион, организация
- Карточка: ФИО, специализация, квалификация, регион, организация

**Реестр судей** (`/public/judges`):
- Список с фильтрацией: поиск, вид спорта, регион
- Карточка: ФИО, виды спорта (несколько), категория, регион

**Реестр организаций** (`/public/organizations`):
- Список с фильтрацией: тип, вид спорта, регион
- Карточка: название, тип, вид спорта, регион, руководитель, аккредитация

**Реестр мероприятий** (`/public/events`):
- Список с фильтрацией: тип, вид спорта
- Карточка: название, даты, тип, вид спорта, место, возраст
- Детальная страница (`/public/events/:id`): полная информация, участники, результаты

**Реестр объектов** (`/public/facilities`):
- Список + вид на карте
- Фильтрация: тип, регион, статус
- Карточка: название, тип, адрес, вместимость, статус

**Реестр команд** (`/public/teams`):
- Список сборных команд КР

**Спортивные школы** (`/public/schools`):
- Список образовательных учреждений спортивного профиля

### 5.2 Авторизация и аутентификация

#### 5.2.1 Регистрация

- Эндпоинт: `POST /api/auth/register`
- Поля: логин, email, пароль, ФИО, телефон
- Роль по умолчанию: `ATHLETE`
- Пароль хешируется BCrypt

#### 5.2.2 Вход (логин)

- Эндпоинт: `POST /api/auth/login`
- Вход по логину или email + пароль
- Возвращает: JWT access-токен (15 мин) + refresh-токен (7 дней)
- Фиксируется: IP, время последнего входа
- **Блокировка:** после 5 неудачных попыток — блокировка на 30 минут

#### 5.2.3 Обновление токена

- Эндпоинт: `POST /api/auth/refresh`
- На вход: refresh-токен
- На выход: новый access-токен
- Ротация refresh-токенов

#### 5.2.4 Выход

- Эндпоинт: `POST /api/auth/logout`
- Отзыв refresh-токена

#### 5.2.5 Структура JWT

```json
{
  "sub": "userId",
  "role": "ROLE_NAME",
  "linkedEntityType": "athlete|coach|judge",
  "linkedEntityId": 123,
  "iat": 1711800000,
  "exp": 1711800900
}
```

    │                              │   (5 обязательных)             │
    │                              │                                │
    │                              ├── Если ОК ────────────────────►│
    │                              │   (статус: «Зарегистрирована»)│
    │                              │                                │
    │◄─ Уведомление ──────────────│◄── Автогенерация сертификата ──│
    │   + номер свидетельства     │   СВ-КР-YYYY-XXXXX            │
    │                              │                                │

    │                              │   (статус: «Требует доработки»)│
    │◄─ Уведомление + комментарий │                                │
```

Гражданин/Федерация          Сотрудник ГАФКиС                   Система
    │                              │                                │
    │── Подаёт заявку ────────────►│                                │

    │                              │    (15/20/30 раб. дней)       │
    │                              │                                │

    │                              ├── Отклонена ──────────────────►│

    │                              │── Голосование                  │
    │◄─ Результат ────────────────│                                │
```

### 7.4 Жизненный цикл мероприятия

```
Запланировано → Идёт → Завершено
      │
      └─→ Отменено

Этапы:
1. Создание мероприятия (статус: «Запланировано»)
2. Добавление участников (спортсменов)
3. При наступлении даты: статус «Идёт»
4. Внесение результатов
5. Автоматическая генерация медалей по местам
6. По завершении: статус «Завершено»
```

### 7.5 Управление сертификатами тренеров и судей

```
Автогенерация номера → Отслеживание срока действия → Предупреждение (30 дн.)
                                                      │
                                                      ├── Продление
                                                      └── Аннулирование
```

## 8. Нефункциональные требования

### 8.1 Безопасность

| Требование | Реализация |
|-----------|------------|
| Хеширование паролей | BCrypt |
| Авторизация | JWT (access 15 мин + refresh 7 дн.) |
| Защита от брутфорса | Блокировка после 5 неудачных попыток (30 мин) |
| RBAC | Ролевая модель на уровне методов (@PreAuthorize) |
| CORS | Ограничение допустимых origins |
| CSRF | Отключен (REST API, stateless) |
| Сессии | Stateless (без серверных сессий) |

### 8.2 Файловое хранилище

| Параметр | Значение |
|----------|----------|
| Макс. размер файла | 20 МБ |
| Макс. размер запроса | 25 МБ |
| Допустимые форматы | PDF, JPG, JPEG, PNG, DOCX |
| Хранилище | MinIO (S3-совместимое) |
| Бакет | `esport-documents` |
| Формат ключа | `{entityType}/{entityId}/{uuid}-{filename}` |

### 8.3 Пагинация

| Параметр | Значение |
|----------|----------|
| Размер страницы по умолчанию | 20 записей |
| Параметры запроса | `page`, `size`, `sort` |
| Формат сортировки | `field,asc|desc` |

### 8.4 Аудит

- Фиксация всех операций (CRUD, смена статусов, загрузка документов)
- Поля: userId, userName, action, entityType, entityId, details (JSON), ipAddress, timestamp
- Доступ: SUPERADMIN, ADMIN
- Фильтрация: по действию, пользователю, диапазону дат

### 8.5 Уведомления

- Per-user поток уведомлений
- Типы: смена статуса, загрузка документа, напоминание о дедлайне, системное сообщение
- Отметка прочитано/непрочитано
- Пагинация
- Массовая отметка «прочитано»
- Счётчик непрочитанных

### 8.6 Обработка ошибок

| HTTP-код | Исключение | Описание |
|----------|-----------|----------|
| 400 | `BadRequestException` | Некорректные входные данные |
| 404 | `ResourceNotFoundException` | Ресурс не найден |
| 409 | `DuplicateResourceException` | Нарушение уникальности |
| 401 | `JwtAuthEntryPoint` | Не авторизован |
| 403 | Spring Security | Нет прав доступа |

Формат ответа об ошибке:
```json
{
  "timestamp": "2026-03-30T10:00:00Z",
  "status": 400,
  "error": "Bad Request",
  "message": "Описание ошибки",
  "path": "/api/endpoint"
}
```

## 11. Интеграции

### 11.1 ТУНДУК (Межведомственная интеграция)

- Опциональная верификация данных тренера через систему ТУНДУК
- Флаг `tunduk_verified` в заявке на регистрацию тренера
- На текущем этапе: флаг устанавливается вручную

### 11.2 MinIO (Файловое хранилище)

- S3-совместимое хранилище документов
- Конфигурация: URL, access-key, secret-key, bucket-name
- Загрузка, скачивание, удаление файлов
- Полиморфная привязка к сущностям через entity_type + entity_id
